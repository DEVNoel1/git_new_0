```bash
git init
git remote and origin SSH_REPOs
```

## Création d'une branche

    ```Bash
    git checkout -b NOM_DE_LA_BRANCHE
    ````

pour les bonnes pratiques, on va intégrer la notion de revue de code. Pour cela, on va créer une branche, faire nos modifications, puis faire une Pull Request (PR) pour que quelqu'un d'autre puisse relire notre code avant de l'intégrer à la branche principale (main ou master).
